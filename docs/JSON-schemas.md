# Gobbler JSON Schemas

Below are the definitions of JSON objects used for 
1. configuring Gobbler server instance
2. defining items that can be sent to Gobbler server instance, and 
3. representing Gobbler telemetry items (telemetry generated by Gobbler server representing significant operational events)

---

## 1. Configuration schema

Configuration object is sent to Gobbler instance as the body of `POST /gobbler/pipeline/configure` endpoint.  

```json
{
  "title": "gobblerConfigurationSchema",
  "description": "describes storage mode and other pipeline properties",
  "type": "object",
  "properties": {
    "mode": {"type": "string", "optional": false},
    "outputDir": {"type": "string", "optional": true},
    "accountName": {"type": "string", "optional": true},
    "accountKey": {"type": "string", "optional": true},
    "writerQueueSize": {"type": "integer", "optional": false},
    "writerBatchSize": {"type": "integer", "optional": false},
    "loggerEndpoint": {"type": "string", "optional": true},
    "loggerTypes": {"type": "array", "items": {"type": "string"}, "optional": true},
    "loggerBatchSize": {"type": "integer", "optional": true},
    "loggerFlushInterval": {"type": "string", "optional": true}
  }
}
```

| Field                | Type            | Required | Description |
|----------------------|-----------------|----------|-------------|
| `mode`               | string          | yes      | Storage backend: `"file"` or `"blob"` |
| `outputDir`          | string          | no       | Root directory for file-mode output |
| `accountName`        | string          | no       | Azure Storage account name (blob mode) |
| `accountKey`         | string          | no       | Azure Storage account key (blob mode) |
| `writerQueueSize`    | integer         | yes      | Capacity of writers item queues |
| `writerBatchSize`    | integer         | yes      | Maximum items written per flush |
| `loggerEndpoint`     | string          | no       | Base URL of the logger Gobbler instance. Omit to disable self-logging |
| `loggerTypes`        | array of string | no       | Item type names that must be register with logger Gobbler instance |
| `loggerBatchSize`    | integer         | no       | Maximum log items sent per batch to the logger (defaults to `100`) |
| `loggerFlushInterval`| string          | no       | Go duration string (e.g. `"10s"`) between logger flushes |

**Example — blob mode with logging enabled:**

```json
{
  "mode": "blob",
  "accountName": "mystorageaccount",
  "accountKey": "base64key==",
  "writerQueueSize": 100,
  "writerBatchSize": 50,
  "loggerEndpoint": "http://logger-gobbler:8080",
  "loggerTypes": [
    "gobbler-ingest-event",
    "gobbler-writer-flush",
    "gobbler-writer-error",
    "gobbler-pipeline-event"
  ],
  "loggerBatchSize": 20,
  "loggerFlushInterval": "10s"
}
```

## 2. Item type definition schema

Item definition object is sent to Gobbler instance as the body of `POST /gobbler/definition/add` endpoint, and introduces a new log item type.  

```json
{
  "title": "itemTypeSchema",
  "description": "item (log entry) type description",
  "type": "object",
  "properties": {
    "name": {"type": "string", "minLength": 3, "optional": false},
    "documentation": {"type": "string", "optional": true},
    "folder": {"type": "string", "minLength": 3, "optional": true},
    "latencyMinutes": {"type": "integer", "optional": true},
    "orderedColumns": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "name": {"type": "string"},
          "type": {"type": "string"},
          "optional": {"type": "boolean", "optional": true},
          "defaultValue": {"type": ["string", "number", "boolean"], "optional": true}
        }
      },
      "uniqueItems": true,
      "minItems": 1
    }
  }
}
```

### Top-level fields

| Field            | Type    | Required | Description |
|------------------|---------|----------|-------------|
| `name`           | string  | yes      | Unique identifier for the item type (minimum 3 characters) |
| `documentation`  | string  | no       | Human-readable description |
| `folder`         | string  | no       | Folder/Container name (minimum 3 characters) for items of this type. Type name is used if the field is omitted |
| `latencyMinutes` | integer | no       | Maximum age of an open file or blob before it is rotated (closed and replaced with a new one); defaults to `1` |
| `orderedColumns` | array   | yes      | Ordered list of column definitions (minimum 1) |

### Column fields

| Field          | Type                       | Required | Description |
|----------------|----------------------------|----------|-------------|
| `name`         | string                     | yes      | Column name |
| `type`         | string                     | yes      | One of: `string`, `int`, `real`, `bool`, `datetime`, `timespan`, `dynamic` |
| `optional`     | boolean                    | no       | Whether the field may be absent in an ingest payload (defaults to `false`) |
| `defaultValue` | string \| number \| boolean | no      | Value used when the field is absent and `optional` is `true` |

### Column types reference

| Type       | Description |
|------------|-------------|
| `string`   | UTF-8 text |
| `int`      | 64-bit signed integer |
| `real`     | 64-bit floating point |
| `bool`     | `true` / `false` |
| `datetime` | `"YYYY-MM-DD HH:MM:SS"` or `"YYYY-MM-DD HH:MM:SS.mmm"` |
| `timespan` | Go duration string, e.g. `"1h30m"` |
| `dynamic`  | Arbitrary JSON object encoded as a string |

## 3. Self-logging item definitions

A running Gobbler instance uses the Gobbler Client SDK to emit its own operational telemetry to a second ("logger") Gobbler instance. This gives operators a queryable, structured record of ingest throughput, flush activity, and internal errors without adding an external observability dependency.

Gobbler operational telemetry is enabled in the configuration - the `loggerEndpoint` and `loggerTypes` are provided. When enabled, Gobbler `ingestion` instance sends telemetry items of the 4 types described below to the Gobbler `logger` instance.  

These 4 item types must be registered with the `logger` instance before the `ingestion` instance is started - each definition must be sent to `logger` using `POST /gobbler/definition/add` endpoint.

---

### `gobbler-ingest-event`

One telemetry item of this type is emitted per each `POST /gobbler/ingest` request.

| Field        | Type   | Optional | Description |
|--------------|--------|----------|-------------|
| `requestId`  | string | yes      | Correlation ID for the request |
| `itemsIn`    | int    | no       | Total items received in the batch |
| `ingested`   | int    | no       | Items accepted by the pipeline |
| `rejected`   | int    | no       | Items in the rejected list |
| `statusCode` | int    | no       | HTTP response status (`200` or `400`) |
| `durationMs` | int    | no       | Handler wall time in milliseconds |

```json
{
  "name": "gobbler-ingest-event",
  "documentation": "One item per POST /gobbler/ingest request handled by Gobbler server.",
  "folder": "gobbler-ingest",
  "latencyMinutes": 5,
  "orderedColumns": [
    { "name": "requestId",  "type": "string", "optional": true  },
    { "name": "itemsIn",    "type": "int",    "optional": false },
    { "name": "ingested",   "type": "int",    "optional": false },
    { "name": "rejected",   "type": "int",    "optional": false },
    { "name": "statusCode", "type": "int",    "optional": false },
    { "name": "durationMs", "type": "int",    "optional": false }
  ]
}
```

---

### `gobbler-writer-flush`

One telemetry items of this type is emitted per each successful writer flush (`FileWriter` or `BlobWriter`).

| Field          | Type   | Optional | Description |
|----------------|--------|----------|-------------|
| `itemType`     | string | no       | Item type being flushed |
| `output`       | string | yes      | Active file path or blob name |
| `itemsFlushed` | int    | no       | Number of CSV lines written in this flush |

```json
{
  "name": "gobbler-writer-flush",
  "documentation": "One item per successful writer flush (FileWriter or BlobWriter).",
  "folder": "gobbler-writer",
  "latencyMinutes": 5,
  "orderedColumns": [
    { "name": "itemType",     "type": "string", "optional": false },
    { "name": "output",       "type": "string", "optional": true  },
    { "name": "itemsFlushed", "type": "int",    "optional": false }
  ]
}
```

---

### `gobbler-writer-error`

One telemetry item of this type is emitted per each writer error (flush, rotate, or open failure).

| Field       | Type   | Optional | Description |
|-------------|--------|----------|-------------|
| `itemType`  | string | no       | Item type whose writer encountered the error |
| `operation` | string | no       | One of: `"flush"`, `"rotate"`, `"open"` |
| `errorMsg`  | string | no       | Error message string |

```json
{
  "name": "gobbler-writer-error",
  "documentation": "One item per writer error (flush, rotate, or open failure).",
  "folder": "gobbler-writer",
  "latencyMinutes": 5,
  "orderedColumns": [
    { "name": "itemType",  "type": "string", "optional": false },
    { "name": "operation", "type": "string", "optional": false },
    { "name": "errorMsg",  "type": "string", "optional": false }
  ]
}
```

---

### `gobbler-pipeline-event`

One telemetry item of this type is emitted per pipeline lifecycle event : `start`, `stop` or `rotate`.

| Field      | Type   | Optional | Description |
|------------|--------|----------|-------------|
| `event`    | string | no       | One of: `"start"`, `"stop"`, `"rotate"` |
| `itemType` | string | yes      | Relevant type name for `"rotate"` events; empty otherwise |

```json
{
  "name": "gobbler-pipeline-event",
  "documentation": "One item per pipeline lifecycle event (start, stop or rotate).",
  "folder": "gobbler-pipeline",
  "latencyMinutes": 5,
  "orderedColumns": [
    { "name": "event",    "type": "string", "optional": false },
    { "name": "itemType", "type": "string", "optional": true  }
  ]
}
```
